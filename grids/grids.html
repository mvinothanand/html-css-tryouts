Defining a Grid

Set display: grid on a container to make it into a grid.

The container transforms into a grid with its children as grid items.

By default it turns into a single column grid with the children appearing one below the other.

Positioning with grid-template-areas

The above grid where the items are placed in reference to the row/column grid lines can be achieved using grid template areas as well.

The elements are given a grid-area name which is then mapped to a cell of the grid.

The mapping of a grid cell to a grid area is defined in the grid container property grid-template-areas.

Example:

grid-template-areas: "header header" "sidebar content" "footer footer"

Just looking at the grid-template-areas definition will provide a view of how the grid is organized.

Rules:

  1. Every cell of the grid should be filled.
  2. If an element has to span across more than one cell, repeate the grid-area name as many times.
  3. Use . for an empty cell.
  4. Areas must be rectangular
  5. Areas can span cells in a single location, but should not be referred in more than one location.

Below grid is organized using grid template areas

grid-template-areas: "one one one" "two five five" "three six seven" "four six seven" "eight eight eight"
